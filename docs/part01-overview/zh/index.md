# Part I 总论与平台观

> 4 章 · 作为全书开篇，负责建立共同语言、平台视角与阅读地图

Part I 的任务，不是立刻进入某个框架、某段代码或某个产品，而是先回答四个更基础的问题：

1. Agent 到底是什么，和 RAG、Copilot、Workflow 有什么边界？
2. 企业为什么真正需要的是平台，而不是一组彼此割裂的智能项目？
3. 什么叫 AI 原生业务系统，它和“在旧系统上加 AI 功能”有什么差别？
4. 后续 55 章为什么按现在这个顺序组织，应该如何阅读？

这四章共同承担的是“卷首总论”的角色。它们不急着进入具体实现，而是追求三件事：

- 先把概念和边界讲清楚；
- 再把企业视角和业务系统视角拉起来；
- 最后给出全书地图和阅读路径。

## 这一部分最适合怎么读

如果你是第一次接触企业级 Agent 平台，建议顺序读完四章，因为它们本来就是一条递进论证链。

如果你已经有一些项目经验，则可以按下面的方式跳读：

| 你当前最关心的问题 | 建议先读 |
|---|---|
| 什么需求值得做 Agent | Ch.01 |
| 团队在争论“平台到底是什么” | Ch.02 |
| 想判断哪些业务适合 AI 原生改造 | Ch.03 |
| 想快速看全书结构和后续路线 | Ch.04 |

Part I 读完之后，读者至少应该建立三种能力：

- 能区分不同类型的大模型系统；
- 能用平台视角重新看企业 AI 项目；
- 能知道后续每个技术主题为什么存在。

带着这三种能力进入 Part II 之后，后面的模型、数据、知识、Runtime、评估、安全和前端章节，就不再是孤立知识点，而会逐渐拼成一张完整的企业级 Agent 平台工程图。
