# 《企业级 Agent 平台工程》中文第一版维护计划

## 一、简介

本书讨论企业如何把大模型、数据平台、知识工程、Agent Runtime、工具生态、评估治理、部署体系、前端交互、安全合规和组织机制组织成可长期运行的平台能力。

全书以 DataAgent 为重要主线，但不把 DataAgent 简化为 NL2SQL，也不把 Agent 简化为聊天应用。写作重点放在四类问题上：

1. 平台边界：哪些能力应沉淀为共享平台，哪些应保留在业务应用中。
2. 工程契约：模型、数据、工具、运行时、前端和治理之间如何建立稳定接口。
3. 证据体系：图表、Trace、评测、参考文献和代码路径是否可复查。
4. 版本诚实：没有可靠材料支撑的案例、鸣谢、项目结果和收益数字不进入正文。

## 三、章节标准

每个中文正文章节应具备以下结构：

1. H1 标题采用 `# 第N章 标题`。
2. 卷首导读依次为 `## 本章摘要`、`## 关键词`、`## 学习目标`、`## 场景引入`。
3. 正文小节按章节号编号，例如第5章使用 `## 5.1`、`### 5.1.1`。
4. 图表必须显式编号并带标题，格式为 `图N-M：标题`、`表N-M：标题`。
5. 图片后必须有独立图题，图题包含来源和 Alt text。
6. 表格前必须有独立表题，表题包含来源。
7. 每章末尾必须有 `## 本章小结` 和 `## 参考文献`。

不再使用 L1/L2/L3 固定模板、生产化清单、上线检查清单或固定虚构公司背景。正文可以讨论真实工程失败模式，但标题和章节骨架应服从正式技术书的编号结构。

## 四、写作风格

写作应学习已发布参考书的组织方式：先给出问题背景，再拆解概念边界、工程机制、图表或表格，最后回到小结与参考文献。避免把章节写成产品宣传、路线图口号或模板化 checklist。

严格禁止以下写法：

1. 用“本章聚焦……把概念边界、架构取舍……”这类重复句式填充摘要。
2. 使用“章节以……为起点，最后回到工程落地、检查项……”这类批量生成句。
3. 使用“山岚集团”或其他固定虚构公司作为统一背景。
4. 编造客户名称、上线规模、ROI、准确率、成本下降比例或真实事故。
5. 用“不是……而是……”连续堆砌观点，造成 AI 口吻。
6. 在没有参考文献或官方文档支撑时断言法规、厂商能力或行业标准。

建议写法：

1. 场景引入使用读者能感知的匿名化复合场景，例如经营分析、客服质检、合同审阅、运维排障、模型路由和权限审计。
2. 概念解释先给边界，再给反例，最后给工程判断。
3. 代码示例只展示能支撑观点的最小片段，并说明与正文架构的关系。
4. 图表用于表达系统边界、状态机、数据流、职责分工或风险控制，不作为装饰。
5. 同一章内避免连续复用相同句式开头。

## 五、图片规范

图片目录以参考书为准，中文正文图片统一向 `docs/images/partN/` 收敛。文件名建议采用稳定英文短名，例如：

```text
docs/images/part2/ch05-model-matrix.svg
docs/images/part5/ch22-run-state-machine.png
docs/images/part7/ch39-dataagent-benchmark-timeline.png
```

图片引用遵循以下要求：

1. Markdown alt 文本必须包含图号和标题，例如 `![图5-1：企业模型矩阵与运行时路由](../images/part2/ch05-model-matrix.svg)`。
2. 图片下一行必须有图题，例如 `*图5-1：企业模型矩阵与运行时路由。来源：本书自绘。Alt text：企业模型矩阵与运行时路由。*`。
3. 同一章内图号从 `图N-1` 连续递增。
4. 跨章节移动图片时，必须同步更新 Markdown 引用和图片检查脚本。
5. 产品截图、界面截图和外部图片必须写清来源；无法确认来源的图片不进入正式版本。

## 六、质量门禁

本地质量门禁以以下命令为准：

```bash
bash scripts/check_all.sh
python -m mkdocs build --strict --clean --site-dir /tmp/enterprise-agent-book-site
```

`scripts/check_all.sh` 至少覆盖：

1. 章节结构和编号检查。
2. 图表编号、标题、来源和 Alt text 检查。
3. 术语和缩写检查。
4. 敏感信息扫描。
5. mini-platform 基础测试。

MkDocs 严格构建用于验证导航、链接、插件和网页电子书渲染。当前中文第一版优先保证中文导航和正文，英文草稿可以暂不进入导航。
